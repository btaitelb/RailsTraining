Column Classes

Bootstrap lets us perform complex layouts simply by creating <div> elements in HTML and assigning them special classes. We can also perform the same layout in pure CSS if we want to keep our classes semantic.

The classes for columns are of the form col-$size-$number, where $size tells us the width of the browser, and $number tells us how many columns that element should take up.

This is all configurable, but by default $size is one of:

  • xs - extra small phones
  • sm - phones and small devices (544px width and up)
  • md - tablets and medium screens (768x and up)
  • lg - desktops (992x and up)
  • xl - extra large desktops (1200px and up)

Since it's a 12-column grid, $number is between 1 and 12.

Mobile First

Bootstrap makes it really easy to build our sites in a mobile first manner. This means instead of having to worry about all different browser sizes (and phone sizes, tablet sizes, etc.) right from the start, we can instead focus on the content of our site and only target the design for the smallest of devices we want to support.

In practice, this means:

  1. Only use col-xs-* or col-sm-* classes across the site and get our content and user flows working
  2. Then test out the site on desktops. It might actually look good enough already to launch.
  3. Add in additional content and layout for larger devices where we have more screen real-estate. Make sure this isn't content that mobile users would want/need.
